[ANK-102] Especificación: Cognitive Scheduler (El Planificador Tokio)
1. Visión General (Abstract)
El Cognitive Scheduler es el corazón del Kernel. Dado que la inferencia de un LLM local es un recurso escaso y bloqueante, el Scheduler gestiona el acceso a este recurso mediante un sistema de Colas de Prioridad y Cambio de Contexto (Context Switching).
Al estar escrito en Rust, el Scheduler operará en un hilo dedicado (background thread) gestionado por el runtime tokio, recibiendo peticiones del servidor gRPC de forma no-bloqueante.
2. Arquitectura de Colas (Rust std::collections & tokio::sync)
El Scheduler mantendrá el estado global protegido mediante Arc<RwLock<T>> para permitir concurrencia segura sin data races:
ready_queue (Cola de Listos): Implementada como un BinaryHeap (Priority Queue nativa de Rust). Ordena los PCBs según su campo priority.
waiting_queue (Cola de Espera): Un HashMap<String, PCB> (PID -> PCB) para procesos en estado WAITING_SYSCALL (esperando I/O o ejecución en Wasm).
Event Channel (MPSC): Un canal Multi-Producer, Single-Consumer (tokio::sync::mpsc). El servidor gRPC (Productor) inyecta nuevos PCBs aquí, y el Scheduler (Consumidor) los procesa.

4. Lógica del Bucle Principal (tokio::select!)
En lugar de un sleep iterativo, Rust nos permite usar macros reactivas. El Scheduler "dormirá" con cero consumo de CPU hasta que ocurra un evento:
code
Rust
// Pseudocódigo Conceptual del Motor
loop {
    tokio::select! {
        // 1. Llegó un nuevo proceso desde el gRPC
        Some(new_pcb) = receiver_channel.recv() => {
            if new_pcb.priority == 10 {
                preempt_current_process();
            }
            ready_queue.push(new_pcb);
        }
        
        // 2. Una Syscall (Wasm/Disco) terminó en el background
        Some(syscall_result) = syscall_channel.recv() => {
            let mut pcb = waiting_queue.remove(syscall_result.pid);
            pcb.registers.insert("last_result", syscall_result.data);
            pcb.state = State::Ready;
            ready_queue.push(pcb);
        }
        
        // 3. La "CPU" (LLM) está libre y hay trabajo en la cola
        _ = engine_is_idle(), if !ready_queue.is_empty() => {
            let pcb = ready_queue.pop().unwrap();
            dispatch_to_cHAL(pcb).await;
        }
    }
}
