### [ANK-904] Especificación: CI/CD Protoc Injection (SRE Firewall)

#### 1. Visión General (Abstract)
El pipeline de integración continua (`pr_check.yml`) falla durante la fase de auditoría de seguridad y tests debido a la falta del compilador de Protobuf (`protoc`). Esto impide que el runner de GitHub Actions compile correctamente el crate `ank-proto`, bloqueando la validación de PRs.

#### 2. Modificaciones Sugeridas
- **workflow (`.github/workflows/pr_check.yml`)**: Inyectar un paso de instalación de dependencias de sistema (`protobuf-compiler`) antes de la inicialización del toolchain de Rust.
- **Optimización**: Asegurar que la instalación se realice de forma atómica y con manejo de errores estricto para evitar pipelines "zombies".

#### 3. Criterios de Aceptación (DoD)
1. El archivo `pr_check.yml` incluye el paso `Install System Dependencies`.
2. El comando de instalación utiliza `sudo apt-get install -y protobuf-compiler`.
3. El pipeline es capaz de superar la fase de compilación de `ank-proto`.

#### 4. Entregables
- `.github/workflows/pr_check.yml` actualizado.
- Registro en `CHANGELOG.md`.
- Commit siguiendo el estándar Conventional Commits.
