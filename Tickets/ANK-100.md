Paso 1: Crear la estructura de carpetas (El Esqueleto)
Abre tu terminal, ubícate en la carpeta donde tienes tu repositorio aegis-ank y ejecuta estos comandos:
code
Bash
# Crear las carpetas base del workspace
mkdir crates
mkdir proto

# Mueve el archivo kernel.proto que creamos antes a la carpeta proto/
# (Asegúrate de que exista proto/kernel.proto)

# Crear los sub-proyectos (crates)
cargo new crates/ank-proto --lib
cargo new crates/ank-core --lib
cargo new crates/ank-server --bin
Tu estructura de carpetas ahora debe verse exactamente así:
code
Text
aegis-ank/
├── proto/
│   └── kernel.proto       <-- El contrato gRPC
├── crates/
│   ├── ank-proto/         <-- Compila el .proto a código Rust
│   ├── ank-core/          <-- El cerebro: Scheduler, cHAL, VCM
│   └── ank-server/        <-- El binario ejecutable (El Daemon)
├── AGENTS.md
└── ARCHITECTURE.md
Paso 2: El Cargo.toml Maestro (Raíz)
En la raíz de aegis-ank/, crea un archivo llamado Cargo.toml. Este archivo le dice a Rust que todos estos subproyectos forman parte de un mismo sistema.
code
Toml[workspace]
members =[
    "crates/ank-proto",
    "crates/ank-core",
    "crates/ank-server"
]
resolver = "2"

[workspace.dependencies]
# Dependencias compartidas para mantener las versiones sincronizadas
tokio = { version = "1.36", features = ["full"] }
tonic = "0.11"
prost = "0.12"
serde = { version = "1.0", features =["derive"] }
serde_json = "1.0"
anyhow = "1.0" # Manejo de errores profesional
Paso 3: Configurar el Crate ank-proto (El Traductor gRPC)
Este crate solo tiene un trabajo: tomar el kernel.proto y convertirlo en código Rust súper rápido.
Edita crates/ank-proto/Cargo.toml:
code
Toml
[package]
name = "ank-proto"
version = "0.1.0"
edition = "2021"

[dependencies]
tonic.workspace = true
prost.workspace = true

[build-dependencies]
tonic-build = "0.11"
Crea un archivo build.rs en crates/ank-proto/ (al lado de su Cargo.toml):
code
Rust
fn main() -> Result<(), Box<dyn std::error::Error>> {
    // Le dice a Cargo que recompile si el archivo .proto cambia
    println!("cargo:rerun-if-changed=../../proto/kernel.proto");
    
    tonic_build::compile_protos("../../proto/kernel.proto")?;
    Ok(())
}
Reemplaza el contenido de crates/ank-proto/src/lib.rs con:
code
Rust
// Exporta el código generado por tonic-build
pub mod v1 {
    tonic::include_proto!("ank.v1");
}
Paso 4: Configurar ank-core (El Motor)
Aquí vivirá el Scheduler, el cHAL y la lógica nativa.
Edita crates/ank-core/Cargo.toml:
code
Toml
[package]
name = "ank-core"
version = "0.1.0"
edition = "2021"

[dependencies]
ank-proto = { path = "../ank-proto" }
tokio.workspace = true
tonic.workspace = true
serde.workspace = true
serde_json.workspace = true
anyhow.workspace = true

# Inferencia Nativa (Llama.cpp)
llama-cpp-2 = "0.3.0"

# Base de Datos Vectorial Embebida (VCM)
lancedb = "0.4.0"

# Para crear IDs únicos
uuid = { version = "1.8", features =["v4"] }
Paso 5: Configurar ank-server (El Ejecutable)
Edita crates/ank-server/Cargo.toml:
code
Toml
[package]
name = "ank-server"
version = "0.1.0"
edition = "2021"[dependencies]
ank-core = { path = "../ank-core" }
ank-proto = { path = "../ank-proto" }
tokio.workspace = true
tonic.workspace = true
anyhow.workspace = true
tracing = "0.1" # Logging de alto rendimiento
tracing-subscriber = "0.3"
⚠️ Requisitos del Sistema (Cuidado aquí)
Para que Rust pueda compilar llama-cpp-2 (que está en C++) y el servidor gRPC, tu sistema necesita dos cosas instaladas:
El compilador de Protobuf (protoc):
Windows: Bájalo de github.com/protocolbuffers/protobuf o usa winget install protoc.
Linux: sudo apt install protobuf-compiler.
Mac: brew install protobuf.
Un compilador de C++ y CMake: (Para que Rust pueda compilar los motores nativos de IA).
Windows: Visual Studio Build Tools (con soporte C++) y CMake.
Linux: sudo apt install build-essential cmake.
El Test Final de la Fundación
Una vez que tengas los archivos en su lugar, corre este comando desde la raíz del proyecto (aegis-ank/):
code
Bash
cargo check
Si todo descarga y arroja un mensaje verde al final (sin errores), acabamos de fundar oficialmente el Kernel.
